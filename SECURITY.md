# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any misfitdev repository, please report it responsibly.

**Do not open a public issue.**

Instead, please use [GitHub Security Advisories](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) to privately report the vulnerability in the affected repository.

If the repository does not have private vulnerability reporting enabled, email **security@misfitdev.com** with:

- A description of the vulnerability
- Steps to reproduce
- Affected versions or branches
- Any potential impact

## Response Timeline

- **Acknowledgement:** within 3 business days
- **Initial assessment:** within 7 business days
- **Fix or mitigation:** depends on severity, but we aim for 30 days for critical issues

## Supported Versions

Unless otherwise noted in a specific repository, only the latest release is supported with security updates.

## Disclosure

We follow coordinated disclosure. We ask that you give us reasonable time to address the issue before making it public.
