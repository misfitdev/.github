# Contributing

Thanks for your interest in contributing to misfitdev projects.

## Getting Started

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Run tests and linters locally before pushing
5. Open a pull request

## Pull Requests

- Keep PRs focused on a single change
- Write a clear description of what and why
- Link any related issues
- Ensure CI passes before requesting review

## Issues

- Check existing issues before opening a new one
- Use the provided issue templates when available
- Include enough detail to reproduce bugs

## Code Style

Follow the conventions established in each repository. When in doubt, match the surrounding code.

## Commit Messages

Write clear, concise commit messages that explain why the change was made. Use imperative mood (e.g., "Add feature" not "Added feature").
